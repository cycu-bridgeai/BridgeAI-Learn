有時候你就是只想試一個想法。

不是要交作業、不是要跑完整專案，就只是想確認「這樣寫會不會動」。但為了這幾行 code，要開 VS Code、設好編譯器、建資料夾……光是準備環境就沒力氣寫 code 了。

BridgeAI 的**練習場**就是為了這種時候存在的。開瀏覽器、進去、直接寫，不裝任何東西。

---

## 1. 四種語言，一鍵切換

練習場支援 **C、C++、Python、Java** 四種語言，直接從工具列點選，不用自己設定編譯器或直譯器。

切換語言的時候，如果你的編輯區已經有程式碼，它會問你要不要用新語言的範例取代——不想換就取消，繼續寫你自己的版本。每種語言都有現成的 starter code，至少 `main` 已經幫你開好了，不用從空白頁面開始。

## 2. 選題目，在練習場解真正的程式題

進練習場預設是「自由模式」——空白編輯器，自己決定寫什麼，下方可以填測試資料當 stdin，跑完在輸出區看結果。適合隨手驗證語法或拆出一小段邏輯單獨測試。

但如果你想練真正的程式題，工具列旁有個**選擇題目**按鈕，點下去會跳出題目選擇器。可以用關鍵字搜尋、篩**難度**（簡單 / 中等 / 困難）、篩**標籤**（排序、遞迴、資料結構⋯⋯），找到想練的題目點進去，左側出現題目描述，右側是編輯器。

寫完提交，不是只看輸出——判題機會拿你的 code 跑完整測試資料，回傳 **OK** 或 **WA**，跟正式作業的評分邏輯一樣。想換題的話點**換題**，回到空白環境就點 **× 清除題目**。

## 3. AI 虛擬碼助手：搭配練習場用，邏輯卡住不用乾等

工具列右側有個 **AI 輔助**按鈕，可以直接在練習場裡開啟 AI 虛擬碼助手。

這個搭配的最大好處是「想到就能試」——在虛擬碼面板把邏輯整理好、插入編輯器之後，馬上就能在同一個頁面執行，不用複製到別的地方去跑。我自己最常在這個流程上卡：邏輯想好了、插進去了，但一執行輸出不對，這時候又可以回到虛擬碼面板調整，幾個來回就把程式理通了。

AI 虛擬碼助手的詳細操作方式（怎麼逐行描述、拖曳排序、插入編輯器）可以看[這篇文章](/blog/pseudo-code)，這邊就不重複了。

## 4. 字體太小？A+ 調大就好

這是個小細節，但考試前緊張的時候特別有感。工具列有 **A+ / A / A-** 三個按鈕，直接調整編輯器的字體大小，不用去找設定頁面。眼睛累的時候放大一點，要看更多程式碼的時候縮回來。

---

## 什麼時候用練習場最合適？

- 上課遇到不確定的語法，想快速驗證
- 作業做到一半，想拆出一個小片段單獨測試邏輯
- 剛開始學一門語言，想試試 Hello World 或基本結構
- 想練題目但不想進作業系統，直接選題、解題、看判題結果
- 懶得開本機 IDE，能動就好

練習場分兩種模式：自由模式純粹跑 code，題目模式可以選真正的程式題練，判題結果跟作業一樣嚴格。需要正式交作業的話還是得去作業那邊，但當成刷題的輕量環境，我覺得它設計得剛好。
